# 피복 신청·승인 관리 시스템

여러 사업소(현장)를 관리하는 본사가 사업소별 계정을 발급하고, 각 사업소는 **피복신청서**
서식 그대로 신청서를 작성해 본사로 제출하면, 본사가 승인/반려로 **결재**하는 브라우저 기반
도구입니다. 승인된 신청은 **지급이력대장**(직원별·품목별 최종 지급일)에 자동으로 반영됩니다.
정적 파일(`index.html`)과 하나의 Cloudflare Worker(`server/`)만으로 동작하며, GitHub
Pages로 배포할 수 있습니다.

## 주요 기능

- **본사 관리자** — 사업소별 아이디/비밀번호 발급, 결재 대기함에서 사업소 신청서를
  상세 확인 후 승인/반려(반려 사유 입력), 전체 현황 통계 대시보드(사업소 수·대기중 신청·
  이번달 승인 금액), 사업소별 신청 이력·지급이력대장 조회
- **사업소 계정** — "피복신청서" 서식 그대로 입력(성명/직급/품목/규격/수량/단가/금액 자동
  계산/지급사유), 엑셀 서식 다운로드·업로드로 대량 입력, 본사로 제출, 제출 이력과 승인/반려
  상태 확인, 본인 사업소의 지급이력대장(직원×품목 최종 지급일) 조회 및 인쇄

## 사용 방법

먼저 `server/README.md`를 따라 백엔드(Cloudflare Worker)를 배포하고, `index.html` 상단의
`API_BASE` 값을 배포된 Worker 주소로 바꾸세요. 이후 GitHub Pages로 배포하려면 저장소 설정에서
Pages 소스를 이 브랜치의 루트(`/`)로 지정하세요.

로그인한 사업소 계정의 입력 데이터는 이 저장소의 `data/` 폴더(비공개 API를 통해서만 접근 가능)에
보관되며, 브라우저에는 로그인 세션 토큰만 저장됩니다.

## 데이터 구조

- `data/users.json` — 계정 목록 (본사 관리자 1명 + 사업소별 계정)
- `data/sites/{siteId}/requests.json` — 사업소별 피복신청서 제출 이력 (상태: 대기/승인/반려)
- `data/sites/{siteId}/issues.json` — 승인된 신청에서 자동 생성되는 지급이력(직원·품목·지급일)
